# 网安线上通知书 · 落地蓝本

> 本文档基于 UI 设计稿 `网安线上通知书.sketch` 整理，作为「从零重建」的开发蓝本。
> 内容包含：技术选型结论、工程目录、**设计令牌（Design Tokens）**、**组件清单**、**页面拆分**、素材与字体清单、后端接口清单、实施路线。
> 旧 `demo/` 目录为演示版本，**将完全弃用**；本目录 `webapp/` 为后期正式上线代码目录。

## 1. 技术选型结论

| 层 | 选型 | 说明 |
|----|------|------|
| 框架 | **Vue 3 + TypeScript** | 模板贴近设计稿结构，D2C/MCP 出码改动少，小团队上手快 |
| 构建 | **Vite** | 快、生态成熟，产物为静态文件 |
| 样式 | **UnoCSS（原子化）+ CSS 变量承载 tokens** | 颜色/间距全部走 token，禁止写死字面量 |
| 令牌 | **Style Dictionary**（可后置） | 设计变量 → `tokens.css` 单一事实来源 |
| 路由 | **vue-router**（6 个页面为流程式路由） | |
| 状态 | **Pinia** | 跨页面共享学生信息、核验状态 |
| 部署 | **静态托管**（GitHub Pages / Vercel / 对象存储） | |
| 后端 | **Serverless 轻后端**（仅 4 类敏感操作，见 §8） | 无需自建传统服务器 |

## 3. 设计令牌 · Design Tokens

### 3.1 颜色 Color

| Token | 值 | 用途 |
|-------|----|------|
| `--color-brand` | `#0050B5` | 网安蓝 / 主色（按钮、标题、品牌） |
| `--color-brand-deep` | `#0A4EA1` | 深蓝（渐变深端） |
| `--color-brand-light` | `#4379BB` | 辅助蓝（渐变浅端 / 次要） |
| `--color-paper` | `#FFF3D4` | 通知书暖色卡片底 |
| `--color-paper-deep` | `#FFECC4` | 暖色卡片深端 |
| `--color-line` | `#CCCCCC` | 分隔线 / 占位 |
| `--color-bg` | `#FFFFFF` | 页面背景 |
| `--color-text` | `#1A1A1A` | 正文（建议值，设计稿正文近黑） |
| `--color-text-invert` | `#FFFFFF` | 反白文字（蓝底上） |

> 蓝色渐变建议：`linear-gradient(180deg, #0A4EA1 0%, #0050B5 100%)`。

### 3.2 字体 Font Family

| Token | 字体 | 用途 | 商用 |
|-------|------|------|------|
| `--font-title` | 阿里妈妈数黑体 `AlimamaShuHeiTi` | 标题/正文大字 | ✅ 免费商用 |
| `--font-button` | 阿里巴巴普惠体 `AlibabaPuHuiTi` | 按钮文字 | ✅ 免费商用 |
| `--font-serif` | 文源宋体 `WenYuanSerifSC` / 思源宋体 | 录取寄语正文 | ✅ 开源 |
| `--font-mono` | 思源黑/宋 `SourceHan*` | 输入占位、二进制底纹、小字 | ✅ 开源 |

## 4. 视觉规律（贯穿全站）

## 6. 页面拆分（6 个画板）

> 统一尺寸 **393 × 852**；所有页面含 `StatusBar` / `BrandHeader`。

### 6.1 欢迎 `WelcomeView`
- **结构**：圆形大 logo（170×170）→「2026级新生」分隔标 →「欢迎，新同学！」(H1 36px) → 录取寄语（宋体 16px）→ 主按钮「开始身份核验」→ 花纹/底部素材。
- **组件**：`BrandHeader`、`AppButton`。
- **数据**：学生姓名（动态）。来源见 §8。
- **交互**：点按钮 → 路由到核验页。

### 6.2 核验 `VerifyView`
- **结构**：`SectionTitle`「身份核验」→ 卡片(学生姓名/身份证号 两个 `AppInput`)→ 主按钮「去认证」→「Hello new student」装饰字 + 底部素材 `6.png`。
- **组件**：`SectionTitle`、`AppCard`、`AppInput`、`AppButton`、`BinaryBackdrop`。
- **数据/逻辑**：**实名核验必须走后端**（§8 `/verify`）；前端只收集并提交。
- **交互**：提交 → 后端核验 → 成功跳通知书页。

### 6.3 通知书 `NoticeView`
- **结构**：整张通知书图 `通知书-07.jpg`(357×502) →「*长按保存通知书」提示(9px)→ 底部「生成合影」「查看报道说明」按钮。
- **组件**：`NoticePaper`、`AppButton`。
- **关键决策**：设计稿通知书是**整图**，无动态文字层。
  - 方案 A（推荐）：让 UI 拆出「底图 + 可变文字层（姓名/录取编号/日期）」，前端用绝对定位叠字。
  - 方案 B：前端用 canvas 在底图上绘制动态文字后导出。
- **数据**：录取数据（姓名/编号/日期）来源见 §8。
- **交互**：长按保存图片；「生成合影」→ 合影加载页；「查看报道说明」→ 注册页。

### 6.4 合影加载页 `PhotoLoadingView`
- **结构**：「镜头正在就位 / 校园合影加载中」(H3 24px) + `LoadingDots` + 「LOADING...」(20px)。
- **组件**：`LoadingDots`、`BinaryBackdrop`。
- **数据**：**纯前端过渡动画**，无后端。
- **交互**：合影合成完成（或定时）→ 自动进入合影页。

### 6.5 合影 `PhotoView`
- **结构**：`PolaroidFrame` 拍立得相框（合成图）→「我在网安等你」+「X同学·2026级新生」→「下载合影图片」「分享到朋友圈/小红书」按钮 + 底部素材。
- **组件**：`PolaroidFrame`、`AppButton`。
- **数据/逻辑**：合影 `useCanvasMerge` **前端 canvas 合成**；下载前端完成；微信官方分享签名需后端（§8 `/wx-sign`），仅「存图自发」则无需后端。
- **交互**：上传/拍照 → 合成 → 下载/分享。

### 6.6 注册 `RegisterView`
- **结构**：`SectionTitle`「线上注册与到校报到」→ 三个 `TimelineItem`（线上注册时间 / 到校报到时间 / 报到地点）→ 主按钮「前往线上注册」。
- **组件**：`SectionTitle`、`TimelineItem`、`AppButton`、`BinaryBackdrop`。
- **数据**：时间地点可作为**静态配置**（如无个性化需求）。
- **交互**：点按钮 → 跳转外部注册链接。

## 7. PC 端适配策略（设计稿缺失项）

设计稿仅有 393×852 移动端。上线若需 PC 访问，推荐：
- **方案 A（推荐）**：PC 上以「移动端画布居中 + 两侧品牌留白/背景」呈现（即手机壳预览式），开发成本最低。
- **方案 B**：请 UI 补 PC 画板，前端做真正的响应式分栏（成本最高）。
- 决策前需与 UI / 业务方确认是否需要 PC 端。

## 8. 后端接口清单（Serverless 轻后端）

> 前端静态部署，仅以下敏感/数据操作走云函数。其余全部前端完成。

| 接口 | 用途 | 为何必须后端 | 对应页面 |
|------|------|--------------|----------|
| `POST /api/verify` | 实名/人脸核验代理 | 第三方实名 API 密钥不能暴露在前端 | 核验 |
| `GET /api/student?token=` | 下发学生录取数据（姓名/编号/日期） | 数据来自招生库，需鉴权 | 欢迎/通知书 |
| `POST /api/chain`（可选） | 区块链存证/查证 | 上链私钥签名必须服务端 | 通知书 |
| `GET /api/wx-sign`（可选） | 微信 JS-SDK 分享签名 | 需 appsecret | 合影 |

> 若定位为「纯典礼演示」，以上均可省略，做成纯静态站。

| `素材底部.png` / `花纹.png` / `1.png` / `6.png` | 装饰素材 |
| `W020260608…png` | 通知书页背景 |

> 导出时统一 `@2x/@3x`，命名规范化（如 `notice-paper.jpg`、`logo-white.png`）。

### 9.2 字体落地注意

1. **子集化**：按实际用到的汉字生成子集（`fontmin` / `glyphhanger`），单字体可从 几 MB 降到几十 KB。
2. **按字重精简**：只保留用到的字重（数黑体 Regular、普惠体 Medium/Regular）。
3. **超大标题**：如「欢迎，新同学！」可考虑转 SVG，避免为单句加载整套字体。
4. **版权确认**：阿里系/思源/文源均可商用，落地前留存授权说明。
