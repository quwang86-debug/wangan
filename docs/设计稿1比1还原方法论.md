## 0. 核心范式：绝对定位还原固定画板

设计稿是 **393×852** 的移动端固定画板，元素均为绝对坐标。还原时：

- 视图根容器：`position: relative; width: var(--design-width); height: var(--design-height); overflow: hidden;`，背景填整屏底色（多数页是 `#0050B5` 全屏蓝）。
- 每个图层用 `position: absolute; left/top/width/height` 按设计稿坐标 **原样** 写 px（画板 393 宽，phone-shell 桌面端正好 393×852，1:1）。
- **z-index 按绘制顺序**：提取脚本输出的列表「自上而下 = 由底到顶」，越靠后越在上层。逐层 +1 设 z-index，避免图层穿透。

## 2. 易踩的坑（务必逐条检查）

1. **填充级不透明度 ≠ 图层不透明度**
   透明度有两处：`layer.style.contextSettings.opacity`（图层级）和 `style.fills[i].contextSettings.opacity`（**填充级**，极易漏）。
   例：欢迎页底部装饰图 fill 透明度 0.2、花纹 0.9、按钮渐变 0.8。脚本已用「【填充不透明度=…】」「【图层不透明度=…】」标出，必须照搬。
   - 图片填充透明度 → CSS `opacity`；按钮/形状渐变填充透明度 → 用伪元素承载渐变并设 `opacity`，避免连同文字一起变淡。

2. **渐变方向**：Sketch `from`/`to` 是归一化向量。`from` 为停止点 0 的位置，`to` 为停止点 1。
   - 例 `from{1,0.5}->to{0,0.5}` = 右(实)→左(透)，CSS 是 `linear-gradient(to left, 实, 透)`（脚本给的「CSS≈XXXdeg」是忽略宽高比的近似，方向务必按 from/to 再核一遍）。
   - 半透明停止点会让**背景色透上来**（设计意图），保留 alpha。

3. **文本 paragraphStyle 取值路径**：在 `attributedString.attributes[i].attributes.paragraphStyle.attributes` 里，**不是** font 描述符下的那个（那个常为空 {}）。关键字段：
   - `alignment`：0左 1右 2居中 **3两端对齐(justify)**
   - `maximumLineHeight`/`minimumLineHeight`：固定行高(px)；都为 None 时才用字体默认行高（本项目文源宋体默认≈1.448）
   - `kerning`：字距(px)，0 就别加 letter-spacing
   - 颜色在 `MSAttributedStringColorAttribute`

4. **字体名前缀 `0.905;TKSW;FontName`**：开头的小数是 **横向缩放(仅文本溢出截断时才生效)**，正常排版**不要**当作 `scaleX` 应用，否则换行行数会错（欢迎页正文若误用 0.905 会变 3 行，正确是 4 行）。

5. **字体平滑**：`reset.css` 里**不要**用 `-webkit-font-smoothing: antialiased`（macOS 上会把中文笔画变细变淡显得"偏浅"）。用 `auto`（次像素）更接近设计稿观感。

6. **圆角**：`shapePath` 的圆角看 points 的 `cornerRadius`（脚本输出 cornerRadius 集）；`rectangle` 看 `fixedRadius`。注意角点内缩比例可能 ≠ 直观半径，以脚本数值为准。

7. **白 logo 是空白占位**：`758c…png`（白色 logo）被 213×46 横向蒙版裁切后，因图片 fit 居中、蒙版只露顶部，实际渲染为**空白**。用空占位 `<span>`（保留 11,41,213×46 尺寸位置），不要塞真实 logo。

8. **状态栏**：顶部 9:41 + 信号/Wifi/电量。可直接复用欢迎页的状态栏 DOM/SVG（时间 14px 白、右侧三图标），坐标见各页 spec 的「编组 4」。

6. 保留原有交互（路由跳转 `useStepper`、表单、相机等逻辑）。
